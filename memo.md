# Forwarding Hub 프로젝트 진행 계획 메모

## 1. 인보이스(청구서) 및 견적서 자동화 기능
*   **목적:** 완료된 수출 건에 대하여 화주가 버튼을 클릭하면 청구서를 다운로드(PDF/Excel 등) 받을 수 있도록 구현.
*   **진행 방식:** 
    *   화주/포워더 실무에서 사용하는 **실제 청구서와 견적서 샘플(엑셀/문서)을 제공받아** 해당 양식을 분석.
    *   데이터베이스에 저장된 비용 데이터(Freight, THC, Doc Fee 등)를 매핑하여 실제 문서 폼에 맞게 파일을 생성 및 다운로드하는 로직(서버사이드 생성)을 개발.

## 2. 부킹(Booking) 및 화물 상태(Status) 관리 프로세스
*   **목적:** 스케줄 조회 후 부킹이 확정된 건에 대해 명확한 상태 변화를 추적하고 업데이트.
*   **상태 변경 흐름 (State Machine):**
    1.  `Booking` (부킹 확정)
    2.  `Arrived at Warehouse` (창고 도착 - Gate In)
    3.  `Loaded` (선적 완료 - Loaded on Vessel)
    4.  `Departed / Delayed` (출항 / 지연 안내)
    5.  `Arrived at Destination` (상대국 입항 - Arrived at POD)

## 3. 알림(Notification) 시스템 연동
*   **목적:** 화물 상태가 변경될 때마다 화주에게 실시간으로 알려주어 서비스 만족도 향상.
*   **진행 방식:**
    *   화물 상태(Status)가 위 2번의 흐름대로 변경될 때마다 **카카오톡 알림톡이나 사내 메신저(슬랙/이메일 등)** API를 호출하여 화주에게 진행 상황을 자동 발송하는 로직 추가.
