# Forwarding-Hub 개발 및 패치 내역 메모 (memo.md)

본 문서는 차량 선적 및 포워딩 관리 플랫폼의 최근 개선 작업 내용, 오류 수정 사항 및 연동 규격을 요약 정리한 개발 메모입니다.

---

## 1. 주요 기능 개선 및 신규 구현 사항

### 📂 화주 서류 보관소 개편 (DocumentListPage.tsx)
- **B/L 번호별 아코디언 UI**: 모든 제출 서류를 평면 테이블이 아닌 B/L 번호별 접이식 아코디언 카드로 묶어 표시함으로써 시각적 복잡성을 대폭 개선했습니다. (최신 B/L 1건은 자동 열림)
- **파일명 라벨링 통일**: 물리 디스크의 난해한 파일명 대신, 사용자에게 친숙한 `Commercial Invoice` 및 `Packing List` 이름으로 파일 열람명이 정규화되어 깔끔하게 노출됩니다.
- **문서 종류 열 삭제**: 파일명 열 자체에 파일 아이콘 및 친숙한 서류 유형 이름이 직관적으로 표기되므로, 중복되던 `문서 종류` 컬럼을 삭제하여 화면 가로 공간을 최적화했습니다.

### 📄 실시간 PDF 생성 및 카카오톡 전송 연동
- **Puppeteer 기반 PDF 제작**: 포워더가 대시보드 상단의 **[PDF 전송]**을 누르면, 등록된 차량 리스트 및 바이어 정보를 조합하여 실제 A4 규격의 Commercial Invoice & Packing List 통합 PDF를 서버에 즉시 생성합니다 (`/uploads/pdf/[B/L번호].pdf`).
- **카톡 알림톡 연동**: 생성 완료와 동시에 화주 카카오 계정으로 PDF 링크 단추가 포함된 알림 메시지가 실시간 전송됩니다.
- **쿠키 및 세션 전달(CORS) 해결**: 포트가 다른 프런트엔드(`5173`)와 백엔드(`5000`) 간 세션 정보가 무사히 교환되도록 API 호출 옵션에 `credentials: 'include'` 설정을 보강했습니다.
- **Puppeteer Chrome 실행 오류 완벽 조치**:
  - 리눅스 환경에 `unzip` 명령어가 누락되어 Chrome 압축 해제가 실패하던 현상을 파이썬 내장 `zipfile` 모듈을 이용한 **임시 가상 `unzip` 스크립트**를 작성해 실행 경로(PATH)에 선탑재하는 방식으로 돌파했습니다.
  - `@150.0.7871.24` 규격의 정확한 브라우저 바이너리를 설치하고, 권한 유실로 인한 실행 차단(`EACCES`)을 해소하기 위해 실행 권한 복구(`chmod -R +x`) 처리를 완료했습니다.

### 🔔 알림 토스트 고도화 및 Wiggle(흔들림) 애니메이션
- **업로드 종류별 알림 세분화**: 화주가 사진을 올릴 때 **차량 외관 사진**을 올렸는지 **차대/말소증 서류**를 올렸는지 소켓 페이로드를 분기하여 포워더에게 각각 구별된 문구로 안내합니다.
- **좌우 흔들림 애니메이션**: 알림 토스트에 `@keyframes wiggle-horizontal` 애니메이션 효과를 부여해 2초 간격으로 좌우로 흔들리게 함으로써 사용자가 중요 이벤트를 직관적으로 인지할 수 있도록 유도했습니다.

### 🚗 차량 관리 대시보드 화면 고도화
- **반응형 가변 높이 및 최대 높이 제한**: 차량이 1~3개일 때는 모달창이 여백 없이 콤팩트하게 열리며, **4개 이상일 때부터 스크롤바가 자동 생성(`max-h-[540px]`)**되도록 디자인을 조정했습니다.
- **등록 대수 실시간 카운트**: 모달 헤더의 바이어(BUYER) 정보창 우측에 현재 B/L에 등록된 차량 개수가 실시간 연동되어 표기되는 **Total Vehicles** 영역을 추가했습니다.
- **드래그 무지연 뷰어 및 디자인**: 150% 커진 말소증 확대 뷰어를 마우스로 드래그할 때 React 상태 재렌더링에 의한 랙이 없도록 DOM Ref를 직접 가리켜 Transform을 조작하는 60fps 무지연 코드를 유지했습니다. 또한 활성화된 차량 행(Row)은 진한 적색 테두리(`border-red-500`) 및 옅은 적색 배경으로 강조 표시합니다.
- **전체화물/선적관리 진입 트랜지션**: 메뉴 진입 시 하단에서 부드럽게 솟아오르는 `animate-fade-in-up` 전환 연출을 적용했습니다.

## 2. 데이터베이스(DB) 및 데이터 정규화 패치

### 🗄️ SQL 테이블 마이그레이션 및 저장 기능 개선
- **단가(`price`) 컬럼 신규 추가**: `vehicles` 테이블에 수출 단가용 `price INT NULL` 컬럼을 생성하고, [전체 저장] 시 오류 없이 저장 및 정상 복원되도록 연동 완료했습니다.
- **날짜 규격 정밀 변환**: 프런트엔드의 날짜 피커 등에서 날아오는 ISO Date 형식 문자열(`YYYY-MM-DDTHH:mm:ss.sssZ`)을 DB 저장 형식인 `YYYY-MM-DD`로 자동 파싱해 주입하는 `parseDateForDb` 안전 장치를 도입하여, SQL 형식 불일치 크래시를 원천 해결했습니다.
- **브랜드 자동 매핑**: 차대 ocr에서 읽어온 원시 데이터(`HMC` 등)를 사전 매핑 테이블을 거쳐 한글 브랜드명(`현대` 등)으로 정규화하여 DB에 밀어 넣습니다.
- **구동상태 기본값**: 구동상태(`drivability`)의 기본값을 기존의 `'Running'`에서 `null`로 지정하여 사용자가 직접 검수하고 입력해야만 저장이 완료되도록 유도했습니다.

## 3. 한글 깨짐 방지 다운로드 표준 도입

- **Content-Disposition RFC 5987 표준**: 엑셀 수출(`exportCustomsExcel`) 및 신설된 파일 다운로드(`downloadFile`) 컨트롤러에 UTF-8 디코딩 규격인 **`filename*=UTF-8''[인코딩된파일명]`** 헤더를 지정했습니다.
- **결과**: 브라우저나 OS 기종에 무관하게 한글 깨짐 없이 정상적인 파일명(예: `관세사용_KMTC.xlsx`)으로 복원되어 안전하게 저장됩니다.
