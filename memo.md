# 포워딩 허브 (Forwarding Hub) 프로젝트 진행 상황 메모

## ✅ 최근 완료된 작업 (2026-07-04)

1. **차량 사진 업로드 UI/UX 완전 분리 (투트랙 업로드 시스템)**
   - `[⚡ 외관 사진만 고속 추가]`: 구글 비전 AI(OCR)를 건너뛰고 1초 만에 사진을 미분류함으로 업로드 (비용 절감 및 속도 극대화).
   - `[🖼️ 차량 등록 (AI 자동 매핑)]`: 기존처럼 AI가 말소증/차대번호 각인을 읽고 표에 차량 리스트를 자동 생성.
   - 팝업 완료 메시지도 버튼 성격에 맞게 분리(`⚡ 고속 업로드 완료!` / `🎉 OCR 분석 완료!`).

2. **포워딩 실무 표준 파일 저장 아키텍처 도입**
   - **미분류 사진**: 최초 업로드 시 `uploads/temp/` 폴더로 저장.
   - **분류/배정 완료 사진**: `uploads/{화주명}/{연도}/{월}/{VIN}/` 폴더 구조를 자동 생성하여 물리적으로 이동 및 보관.
   - **한글 파일명 에러 해결**: 업로드 시 파일명을 무작위 난수(`photo_timestamp_random.jpg`)로 자동 변환하여 인코딩 깨짐(엑스박스) 문제 영구 해결.

3. **드래그 앤 드롭 파일 배정 API 구축**
   - 프론트엔드에서 미분류함의 사진을 표의 차량으로 드래그할 경우, 백엔드에서 `temp` 폴더의 파일을 해당 VIN 전용 폴더로 즉각 이동.
   - `vehicles` 테이블의 `condition_photo_url` 컬럼을 `TEXT` 타입으로 확장하여 최대 10장의 사진 URL을 JSON 배열 형태로 안정적으로 저장.

4. **서버 용량 최적화 (자동 청소 크론잡)**
   - 매일 오전 9시, 백그라운드 스케줄러가 `uploads/temp/` 폴더를 검사.
   - 생성된 지 **1주일(7일)이 경과한 미배정 사진 찌꺼기를 서버에서 영구 자동 삭제**.

## 🚀 다음 진행할 작업 (Next Steps)

### 1. 국내 내수용 차량 제원(VIN) 조회 연동 (공공데이터포털)
현재 사용 중인 미국(NHTSA) API는 국내 내수 차량 조회가 불가능합니다. 이를 해결하기 위해 한국 교통 데이터 API를 연동해야 합니다.

* **화주(대표님) 측 준비 사항 (다음 세션 시작 시):**
  1. 공공데이터포털(`data.go.kr`)에 접속 (보안프로그램 충돌 시 모바일 접속 권장 또는 브라우저 캐시 삭제 후 간편로그인 시도)
  2. 로그인 후 **[국토교통부_자동차종합정보 API서비스]** 활용 신청
  3. 마이페이지에서 발급된 **[일반 인증키(Encoding/Decoding)]** 복사 후 채팅창에 전달

* **AI 개발자(Antigravity) 측 작업 예정:**
  1. 전달받은 인증키를 서버 환경변수(`.env`)에 세팅
  2. `vinService.ts`를 수정하여 공공데이터포털로 API Request 발송
  3. 응답받은 XML 데이터를 파싱(`xml2js` 등 활용)하여 제조사, 모델명, 연식 등을 표에 반영

### 2. (보류 중) PDF 자동 생성 로직 
- 인보이스 및 패킹 리스트 데이터를 기반으로 한 실제 PDF 렌더링 백엔드 로직 구현.
- 생성된 PDF 파일을 화주에게 카카오톡(알림톡)으로 자동 전송하는 API 연동.
