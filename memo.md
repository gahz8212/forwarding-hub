# 오늘 진행한 작업 내역 요약 (AI 차량 대시보드 & 서류 OCR 최적화)

## 1. 화주 서류 업로드 실시간 알림 연동
- **웹소켓(Socket.io) 실시간 알림 구현:** 화주가 외부에서 차량 말소증 및 외관 사진을 업로드하면 새로고침 없이 즉시 포워더 화면 우측 하단에 `Toast(토스트) 팝업`이 뜨도록 구현했습니다.
- **다이렉트 대시보드 열기:** 토스트 알림창에 있는 "차량 관리 대시보드 열기" 버튼을 누르면 B/L 번호를 찾아갈 필요 없이 즉시 해당 모달창이 열리게 됩니다.
- **뱃지 자동 갱신 연동:** 화주가 사진을 업로드할 때 모달이 열려있는 상태라도 백그라운드에서 실시간으로 대기 서류 숫자가 증가하며 뱃지가 갱신됩니다.

## 2. OCR (구글 비전 AI) 인식 성능 및 압축 이슈 해결
- **이슈:** 화주가 업로드 시 임시 폴더에 저장할 때 용량을 위해 사진을 '압축(리사이징+화질80%)'하여 저장했는데, 이로 인해 글씨가 작고 얇은 표 형식의 말소증 문서에서 인식률이 떨어지고 차대번호만 간신히 인식되는 문제가 발생했습니다.
- **해결책 (고화질 백업 보존):** 화주가 업로드할 때, 미리보기용 압축 사진과 별개로 **OCR 분석 전용 무손실 원본 백업(`original_...`)**을 잠시 생성해 둡니다. 분석 버튼을 누르면 이 고화질 원본을 가져와서 완벽하게 분석을 돌리도록 수정했습니다.
- **정규식(Regex) 강화:** 표 형식의 데이터에서 줄바꿈이 섞이거나 글자가 멀리 떨어져서 인식되더라도(예: '주행거리'와 'km' 사이의 공백, 불규칙한 '최초등록일' 날짜 포맷 등) 주행거리, 연식, 최초등록일, 차종, 번호판 데이터를 유연하게 모두 잡아내도록 추출 알고리즘을 대폭 강화했습니다.

## 3. 서류 확인 뱃지(Magnifier) 편의성 개선
- **마우스 호버 돋보기:** 뱃지를 클릭했을 때 나오는 서류 썸네일 이미지 위에 마우스를 올리면 **실시간으로 3배 확대**되는 기능을 구현하여, 글씨가 작은 말소증을 쉽게 읽고 비교할 수 있게 되었습니다.
- **서류 지속 유지 기능 (Copy 로직):** 분석 버튼을 누르자마자 뱃지에서 서류가 사라지지 않습니다. [전체 저장] 버튼을 누르기 전까지는 언제든 뱃지를 다시 열어 돋보기로 OCR 결과(차대번호, 연식 등)가 맞게 들어갔는지 사진 원본과 대조해 볼 수 있습니다. 

## 4. [전체 저장] 버튼 API 완벽 구현
- **저장 로직 완료:** 단순히 `alert`만 뜨던 [전체 저장] 버튼에 백엔드 API 연동을 완료했습니다. 표에서 포워더가 직접 수정한 데이터들이 데이터베이스(vehicles 테이블)에 일괄 업데이트됩니다.
- **임시 폴더 자동 정리 로직:** 데이터가 무사히 DB에 저장되면, 그제야 뱃지에 남겨두었던 **분석 완료된 말소증 사진 원본들과 백업본들을 임시 폴더에서 깔끔하게 삭제**하여 서버 용량을 확보하고 뱃지 알림을 사라지게 만듭니다. (미분류된 외관 데미지 사진은 제외)
