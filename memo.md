## 📅 오늘 완료한 작업 (2026-07-08)

### 1. 중복 파일 업로드 차단 및 실시간 동기화 정교화
- **백엔드 글로벌 중복 체크 확장**: 
  - 사진 업로드 시 `uploads/temp` 뿐만 아니라 해당 화주의 영구 저장 폴더(`uploads/[화주명]`)까지 전체 재귀 스캔하여 중복 해시를 대조합니다. 이미 저장 완료된 사진의 재업로드를 완벽히 차단합니다.
  - 화주명 조회 시 B/L 번호나 ID 형태 모두에 대응하도록 `resolvedShipmentId` 매핑 로직을 보완하고, 폴더 경로가 어긋나지 않도록 특수문자 제거 및 공백 트림(`trim()`) 규칙을 백엔드 전체에 통일 적용했습니다.
- **프론트엔드 중복 예외 처리**: 
  - 백엔드에서 중복(`status === 'duplicate'`) 판정을 받은 파일은 UI 상태 배열(`unclassifiedPhotos` 및 `pendingPhotos`) 추가 대상에서 완전히 제외시켰습니다.
  - 업로드 완료 시 중복 파일이 존재할 경우 **`"선택한 이미지가 이미 저장되어 있습니다."`** alert 안내창을 제공합니다.
- **실시간 소켓/뱃지 동기화 조건 제약**: 
  - 실제로 임시 폴더에 신규 저장된 사진 개수(`savedCount`)가 0보다 클 때만 Socket.io 알림 이벤트(`new_shipper_docs_alert`)를 발송합니다.
  - 모두 중복이거나 오류인 파일을 업로드할 경우 포워더 화면의 뱃지가 깜빡이거나 생성되지 않도록 수정했습니다.

### 2. 차량관리 대시보드 저장소 구조 개선 (전체 저장 시)
- **BL 기준 하위 폴더 분기**: 
  - 포워더 대시보드에서 `[전체 저장]` 클릭 시, 임시 폴더(`temp/[BL번호]`)에 있던 `docs` 및 `exterior` 폴더 구조를 그대로 유지하며 영구 폴더(`uploads/[화주명]/[연]/[월]/[BL번호]/[하위폴더]`)로 이동되도록 구조를 개편했습니다.
  - 차량 배정용 임시 VIN 폴더에 들어있던 파일들도 자동으로 `BL번호/exterior/` 하위로 취합 이관되며, DB의 사진 URL 정보도 이에 맞춰 일괄 변환 저장됩니다.

### 3. 상세 사진 뷰어 UI/UX 고도화
- **뷰어 위치 및 크기 통일**: 사진정보 셀 클릭 시 열리는 상세 사진 뷰어 팝업의 초기 렌더링 위치 및 크기를 우측 상단 미분류 사진함과 완전히 일치시켰습니다.
- **겹침 방지**: 사진 상세 뷰어가 열리면 미분류 사진함 드로어가 자동으로 닫히도록 배타적 처리했습니다.
- **핀터레스트 스타일 Masonry 레이아웃 적용**: 상세 사진 뷰어 내부 격자를 기존 16:9 크롭 방식(`aspect-video`)에서 원본 종횡비를 완전히 보존하여 빈틈없이 나열하는 모자이크(`columns-2`) 방식으로 전면 개편했습니다.
- **휴지통 이관 연동**: 사진 뷰어에서 🗑️(휴지통) 버튼 클릭 시 해당 차량에서 제거된 사진이 미분류 사진함으로 정상 즉시 환원되며, 미분류 드로어가 자동 팝업되도록 연결했습니다.

### 4. 포워더/화주 업로드 사진 분리 및 뱃지 필터링
- **파일명 접두사 부여**: 파일 업로드 시 주체(`isForwarder`)를 판별하여 `temp` 폴더 내 저장되는 파일명 앞에 `forwarder_` 또는 `shipper_` 접두사를 붙이도록 로직 고도화.
- **뱃지 및 사진함 필터링**: `getUnclassifiedPhotos` API에서 목록을 읽을 때 `forwarder_` 접두사를 가진 파일은 무시(`continue`)하여, 포워더가 업로드한 사진은 포워더의 "화주가 보낸 서류" 뱃지 카운트와 미분류함에 노출되지 않도록 완벽히 분리.
- **포워더 대행 업로드 지원**: 포워더가 화주를 대신해 문서를 올리고 작업을 진행하는 시나리오를 방해 없이 수행 가능하게 개선.

### 5. OCR 연동 및 인증 환경 최적화
- **OCR 수동 검수 연동 재확인**: 차대번호나 번호판이 명확히 인식되지 않는 테스트용 사진을 올렸을 때만 "대기 중인 사진 검수 및 AI 분석" 모달 창으로 빠지며, 인식 성공 시 즉시 등록되는 기존 OCR 파이프라인의 건재함 확인 및 안내.
- **구글 Vision API 인증 경로 교정**: 윈도우 확장 프로그램 기준 경로(`/mnt/c/...`)로 잘못 기입되어 있던 `.env`의 `GOOGLE_APPLICATION_CREDENTIALS` 환경 변수를, 현재 구동 중인 리눅스(WSL) 환경의 `gcloud` 로컬 키 경로(`/home/gahz/.config/gcloud/application_default_credentials.json`)로 수정하여 OCR 인증 오류를 영구 해결.

## 🔍 중복 파일 검증 시나리오

1. **[초기화] 클릭**: 대시보드 내 초기화 버튼을 클릭하여 임시 폴더를 완전히 비우고 뱃지를 `0`으로 만듭니다.
2. **신규 업로드**: `A.jpg` 사진을 올리면 정상 처리 메시지가 나타나며 뱃지가 `1`로 증가합니다.
3. **중복 업로드 (임시 상태)**: `A.jpg`를 다시 올리면 `"선택한 이미지가 이미 저장되어 있습니다."` alert가 뜨고, 새로고침을 해도 뱃지가 `1`로 유지됩니다.
4. **전체 저장**: `[전체 저장]`을 클릭하면 파일이 영구 폴더로 이관되고 임시 폴더가 비워지므로 뱃지는 `0`이 됩니다.
5. **중복 업로드 (영구 상태)**: 영구 저장된 `A.jpg`를 다시 업로드하면 영구 폴더 내 해시를 대조해 차단하며 `"선택한 이미지가 이미 저장되어 있습니다."` alert가 뜨고 뱃지는 여전히 `0`으로 유지됩니다.
