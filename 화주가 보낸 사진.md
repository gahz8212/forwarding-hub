포워딩 로로(Ro-Ro) 중고차 수출 업무에서 화주가 보낸 사진 더미를 **Node.js와 상용 OCR API를 활용해 차량별로 자동 분류 및 폴더링**하는 전체 프로세스를 CLI 환경에서 구축하고 실행할 수 있도록 작성한 상세 가이드라인입니다.

이 가이드라인을 `README.md` 파일로 저장하여 프로젝트 관리나 기획서로 활용하실 수 있습니다.

# 📑 포워딩 로로 중고차 수출 사진 자동 분류 시스템 구축 가이드

본 문서는 화주가 카카오톡 등으로 전송한 정제되지 않은 중고차 사진 더미(말소증, 차대번호 각인, 차량 외관)를 상용 OCR API를 활용하여 **[차량번호_차대번호]** 형태의 폴더로 자동 분류하고 이동시키는 CLI 기반 시스템의 아키텍처 및 작업 환경 구축 가이드입니다.

## 🎯 1. 최종 목적 및 핵심 메커니즘

### 1.1 최종 목적

* **입력(Input):** 하나의 폴더(`input/`)에 여러 차량의 말소증, 차대번호, 외관 사진 100여 장이 뒤죽박죽 섞여서 들어옴.
* **출력(Output):** 시스템 실행 후, `output/아반떼_12가3456_KMHCT41HPUU123456/`과 같이 **차종_차량번호_차대번호** 단위로 폴더가 자동 생성되고 해당 차량의 사진들만 묶여서 이동됨.

### 1.2 핵심 식별 메커니즘 (데이터 매칭 논리)

1. **마스터 데이터 확보 (말소증):** - 말소등록사실증명서는 국가 규격 양식이므로, OCR 처리 시 '차량번호'와 '차대번호(17자리)'가 동시에 추출됩니다. 이 두 데이터를 하나의 '차량 객체(Key)'로 묶어 마스터 테이블을 형성합니다.
2. **차대번호 각인 사진 검증:**
* 엔진룸이나 문짝에 타각된 차대번호 사진을 OCR로 읽어 17자리 영문+숫자 조합을 추출한 뒤, 마스터 데이터의 차대번호와 비교하여 일치하는 차량 폴더로 분류합니다.


3. **차량 외관 사진 분류:**
* 차량 앞/뒤 사진에 포함된 '번호판' 영역을 OCR로 읽어 차량번호(예: 12가3456)를 추출한 뒤, 마스터 데이터의 차량번호와 매칭하여 해당 차량 폴더로 분류합니다.


4. **차종 판별 (선택 사항):**
* 말소증의 '차명'란(예: 아반떼, 소나타) 텍스트를 추출하여 폴더명의 접두사로 활용합니다.

## 🛠️ 2. 개발 환경 준비 및 CLI 명령어

본 시스템은 자바스크립트 런타임인 **Node.js** 환경에서 실행됩니다. 터미널(CLI)에서 아래 순서대로 명령어를 입력하여 환경을 조성합니다.

### 2.1 프로젝트 폴더 생성 및 초기화

```bash
# 프로젝트를 진행할 디렉토리 생성 및 이동
mkdir lolo-photo-classifier
cd lolo-photo-classifier

# Node.js 프로젝트 초기화 (package.json 생성)
npm init -y

```

### 2.2 필수 종속성 라이브러리 설치

```bash
# 외부 라이브러리 설치
npm install @google-cloud/vision fs-extra chokidar sharp

```

* **`@google-cloud/vision`**: 구글 클라우드에서 제공하는 최상위 수준의 이미지 인식/문서 OCR SDK입니다.
* **`fs-extra`**: Node.js 기본 파일 시스템(`fs`)을 확장하여 폴더 생성(`ensureDir`), 파일 이동(`move`)을 강력하고 단순하게 처리합니다.
* **`chokidar`**: 디렉토리 실시간 감시 라이브러리입니다. 영업팀이 `input/` 폴더에 사진을 넣는 순간을 감지해 이벤트를 발생시킵니다.
* **`sharp`**: 대용량 초고화질 사진을 고속으로 리사이징(용량 압축) 및 흑백 전환하여 API 비용을 아끼고 속도를 높이는 이미지 처리 도구입니다.

## 🔑 3. 상용 OCR API 연동 및 클라우드 설정

정확한 한국어 문서 및 차량 번호판 인식을 위해 **Google Cloud Vision API** 또는 **NAVER CLOVA OCR**을 사용합니다. 가이드는 전 세계 표준인 Google Cloud 기준으로 설명합니다.

### 3.1 Google Cloud Platform(GCP) 인증 키 발급 절차

1. [Google Cloud 콘솔](https://console.cloud.google.com/)에 접속하여 프로젝트를 생성합니다.
2. **API 및 서비스 > 라이브러리** 메뉴에서 "Cloud Vision API"를 검색한 후 [사용] 버튼을 활성화합니다.
3. **IAM 및 관리자 > 서비스 계정** 메뉴로 이동하여 이 시스템 전용 서비스 계정을 생성합니다.
4. 생성된 서비스 계정의 **[키(Key)]** 탭으로 이동하여 [새 키 만들기 -> JSON]을 선택합니다.
5. 다운로드된 `xxxxx-auth-key.json` 파일을 프로젝트 최상위 루트 디렉토리로 이동시키고 파일명을 `google-credentials.json`으로 변경합니다.

### 3.2 CLI 환경 변수 설정 (인증 연동)

Node.js 코드가 실행될 때 구글 인증 키를 자동으로 인식할 수 있도록 시스템 환경 변수를 설정해야 합니다.

* **Windows (Command Prompt):**
```cmd
set GOOGLE_APPLICATION_CREDENTIALS="google-credentials.json"

```


* **Windows (PowerShell):**
```powershell
$env:GOOGLE_APPLICATION_CREDENTIALS="google-credentials.json"

```


* **Mac / Linux (Terminal):**
```bash
export GOOGLE_APPLICATION_CREDENTIALS="google-credentials.json"

```

```text
└── package.json                # 프로젝트 메타데이터 및 의존성 환경 정보

```

## 🎯 5. 데이터 정제 및 정규표현식(Regex) 분석 로직

상용 API가 이미지에서 텍스트를 한 줄 단위 혹은 문단 단위로 추출해 주면, 자바스크립트 내부에서는 컴퓨터가 이해할 수 있도록 정규표현식(Regular Expression)을 이용해 필요한 알맹이만 파싱해야 합니다.

### 5.1 대한민국 자동차 번호판 매칭 정규식

대한민국 자동차 등록번호의 표준 규칙을 필터링합니다.

* **정규식 패턴:** `/\d{2,3}[가-힣]{1}\d{4}/`
* **매칭 예시:** `12가3456` (구형 7자리), `123가4567` (신형 8자리)
* **적용 단계:** 말소증 상단의 등록번호 영역 및 차량 앞/뒤 외관 사진의 번호판 영역 탐색 시 적용.

### 5.2 국제 표준 차대번호(VIN) 매칭 정규식

전 세계 공용인 17자리 차량 고유 식별 번호를 필터링합니다. 단, 오인식 방지를 위해 디지털 서체에서 헷갈리기 쉬운 영문 `I`, `O`, `Q`는 차대번호에 사용되지 않는다는 규칙을 반영합니다.

* **정규식 패턴:** `/[A-HJ-NPR-Z0-9]{17}/`
* **매칭 예시:** `KMHCT41HPUU123456` (알파벳 대문자와 숫자의 17자리 조합)
* **적용 단계:** 말소증 내부의 차대번호 란 및 엔진룸/대시보드에 타각된 철판 각인 사진 탐색 시 적용.

## ⚠️ 6. 실무 오퍼레이터를 위한 예외 처리 및 방어 설계

상용 API를 맹신하면 실제 거친 물류 현장에서 프로그램이 터지거나 멈춥니다. 오퍼레이터의 퇴근 시간을 지키기 위해 시스템 설계 시 반드시 반영해야 하는 실무적 방어 전략입니다.

### 6.1 비동기 대량 처리 (Queue) 및 비용 최적화

* 화주가 올린 원본 사진은 장당 5MB~10MB에 육박합니다. 이를 그대로 구글 서버로 전송하면 네트워크 트래픽 비용과 API 처리 비용이 막대하게 발생합니다.
* **해결책:** 파일이 `input/`에 감지되면, Node.js의 `sharp` 라이브러리를 통해 즉시 가로 해상도 `1920px` 수준의 FHD 크기로 리사이징하고 해상도를 낮춘 임시 파일로 변환합니다. 또한 색상 정보를 지우고 **흑백(Grayscale) 처리**를 거친 뒤 API로 전송합니다. 텍스트 인식률은 동일하게 유지되면서 속도는 5배 빨라지고 비용은 대폭 절감됩니다.

### 6.2 일치 불일치 크로스 체크 (샌드위치 방어막)

* **비정상 상황:** 말소증에 적힌 차량번호는 `12가3456`인데, 외관 사진 속 번호판은 `12가3458`로 찍혀 있는 경우 (화주가 사진을 섞어서 잘못 보낸 경우).
* **해결책:** 시스템 내부 조건문에서 `말소증_차량번호 === 외관_번호판`을 비교하여 불일치할 경우, 정규식에 걸렸더라도 해당 차량 폴더로 이동시키지 말고 즉시 `output/미분류_손상데이터/` 폴더로 격리하고 CLI 터미널 창에 "[위험] 차량번호 불일치 건 발생 - 파일명: IMG_012.jpg"라는 경고 로그를 빨간색으로 출력하도록 설계해야 합니다.

### 6.3 무조건적인 격리 조치 (Graceful Failure)

* 초점이 흐리거나, 야간에 찍혀 빛이 번진 차대번호 각인 사진은 구글 AI도 텍스트를 읽지 못해 Null(빈 값)을 리턴합니다.
* **해결책:** 프로그램이 에러를 뿜으며 멈추지 않도록 모든 OCR 단계를 `try-catch` 구문으로 감싸야 합니다. 식별 키를 찾지 못한 사진은 조용히 `output/미분류_손상데이터/` 폴더로 이동시킵니다. 오퍼레이터는 퇴근 전 미분류 폴더에 남은 2~3장의 사진만 수동으로 마우스 드래그하여 넣어주면 업무가 종료됩니다.

## 🚀 7. 시스템 실행 및 모니터링 (CLI 명령어)

모든 아키텍처와 환경 설정이 완료된 후, 터미널에서 시스템을 구동하고 모니터링하는 명령어입니다.

### 7.1 수동 배치 실행 (모여있는 사진 한 번에 처리)

```bash
# input 폴더에 쌓여있는 사진들을 일괄적으로 분석하여 분리 시작
node index.js

```

### 7.2 백그라운드 상시 감시 실행 (영업팀 폴더 연동 시)

서버나 개인 컴퓨터를 켜두고 영업팀이 폴더에 사진을 넣을 때마다 실시간으로 낚아채서 분류하게 하려면 Node.js 상시 프로세스 관리 도구인 `pm2`를 CLI에 도입합니다.

```bash
# PM2 글로벌 설치
npm install pm2 -g

# index.js를 백그라운드 상시 감시 모드로 실행
pm2 start index.js --name "ro-ro-classifier"

# 실시간 분류 로그 및 에러 상황 터미널 모니터링
pm2 logs ro-ro-classifier

```

이 상태를 구축해 두면 오퍼레이터는 터미널을 건드릴 필요 없이, 컴퓨터가 켜져 있는 동안 자동으로 폴더들이 알아서 생성되고 쪼개지는 광경을 보게 됩니다.
