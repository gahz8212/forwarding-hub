# 🚀 로로선 중고차 사진 자동 분류(OCR) 시스템 테스트 가이드

지금까지 구축한 시스템(프론트엔드 ZIP 업로드 + 백엔드 Sharp 압축 + 구글 Vision API)이 완벽하게 동작하는지 처음부터 끝까지 테스트하기 위한 정확한 가이드라인입니다.

---

## 1단계: 구글 클라우드 비전(Vision API) 세팅 및 키 발급

우리 코드는 완성되었지만, 구글 서버에 접속하기 위한 '입장권(JSON 키)'이 없으면 에러가 발생합니다.

1. **GCP 콘솔 접속:** [Google Cloud Console](https://console.cloud.google.com/)에 구글 계정으로 로그인합니다.
2. **새 프로젝트 생성:** 상단에서 새 프로젝트를 만듭니다. (예: `forwarding-ocr`)
3. **결제 수단 등록:** 무료 혜택(월 1,000장)을 받기 위해서라도 최초 1회 신용카드 등록이 필요합니다.
4. **API 활성화:** 상단 검색창에 **"Cloud Vision API"**를 검색한 뒤 **[사용(Enable)]** 버튼을 누릅니다.
5. **인증 키 발급:**
   * 왼쪽 메뉴: `IAM 및 관리자` > `서비스 계정`
   * `[서비스 계정 만들기]` 클릭 후 이름 대충 적고 완료.
   * 만들어진 계정을 클릭하고 상단의 `[키]` 탭으로 이동.
   * `[키 추가]` > `[새 키 만들기]` > **JSON** 선택하여 다운로드.
6. **키 파일 이동:** 다운로드된 `.json` 파일을 포워딩 허브의 `backend` 폴더 안에 넣고, 이름을 알아보기 쉽게 `google-credentials.json`으로 변경합니다.

---

## 2단계: 환경 변수(인증 키) 설정 및 서버 실행

터미널을 열고 백엔드 서버를 구동할 때, 방금 받은 구글 키를 인식시켜야 합니다.

**[백엔드 서버 실행 방법 (backend 폴더에서)]**
\`\`\`bash
# 1. 터미널 환경변수에 키 경로 등록 (리눅스/맥 기준)
export GOOGLE_APPLICATION_CREDENTIALS="./google-credentials.json"

# (윈도우 PowerShell인 경우)
# $env:GOOGLE_APPLICATION_CREDENTIALS="./google-credentials.json"

# 2. 백엔드 서버 실행
npm run dev
\`\`\`

**[프론트엔드 서버 실행 방법 (새 터미널 열고 frontend 폴더에서)]**
\`\`\`bash
npm run dev
\`\`\`

## 3단계: 테스트용 데이터(사진/ZIP) 준비

실제 상황을 모방하기 위해 테스트용 데이터를 준비합니다. 구글에서 샘플 사진을 몇 장 다운로드합니다.
* **말소증:** 구글 이미지 검색에서 "자동차 말소등록사실증명서" 검색 후 사진 1장 캡처.
* **번호판:** 구글에서 "한국 자동차 번호판" 검색 후 12가3456 형태가 선명한 사진 1장 캡처.
* **차대번호:** "자동차 차대번호 각인" 검색 후 영어+숫자 조합 타각 사진 1장 캡처.

이 사진들을 하나의 폴더에 넣고 **`테스트.zip`**으로 압축합니다.

---

## 4단계: 프론트엔드에서 최종 테스트 진행

1. 인터넷 브라우저를 열고 프론트엔드 로컬 주소(예: `http://localhost:5173`)로 접속합니다.
2. **로로선 차량 관리 대시보드 모달** 창을 띄웁니다.
3. 우측 상단의 **[차량 사진 일괄 업로드]** 버튼을 클릭합니다.
4. 아까 만들어둔 **`테스트.zip`** 파일을 선택하여 엽니다.
5. 버튼이 `OCR 분석 중...`으로 변하며 백엔드로 전송됩니다.
6. **[결과 확인 1]** 완료 알림창(Alert)이 뜨고, `F12`를 눌러 개발자 도구의 콘솔(Console) 탭을 보면 사진별로 추출된 차대번호와 차량번호가 뜹니다.
7. **[결과 확인 2]** 데이터베이스 도구(DBeaver, MySQL Workbench 등)를 열어 `vehicles` 테이블을 조회해 봅니다. 차대번호(vin), 상태(Yard In), 사진 경로 등이 제대로 저장되었는지 확인합니다!
